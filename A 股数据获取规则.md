## 🚫 核心原则（不可违反）

### 第一条：禁止编造数据 🔴

```
❌ 绝对禁止猜测、估算、编造任何股票数据
❌ 绝对禁止使用缓存数据冒充实时数据
❌ 绝对禁止在数据缺失时自行填充
❌ 绝对禁止修改原始数据

✅ 必须标注数据来源和获取时间
✅ 必须如实报告数据获取失败
✅ 必须使用真实获取的数据
```

```
1️⃣ 东方财富 (AkShare) - 主数据源

   ↓ (失败/限流时切换)
3️⃣ 腾讯财经 HTTP API - 第三数据源
```

## 🔧 标准数据获取流程

```python
def get_stock_data(code):
    """
    【系统级函数】获取股票数据
    所有机器人必须使用此标准流程
    """
    
    # 1. 尝试主数据源（东方财富）
    try:
        data = akshare_get(code)
        if validate_data(data):
            return data, '东方财富'
    except Exception as e:
        log_error(f"东方财富失败：{e}")
        time.sleep(0.5)
    
    # 2. 尝试备用数据源（新浪财经）
    try:
        data = sina_http_get(code, delay=0.3)
        if validate_data(data):
            return data, '新浪财经'
    except Exception as e:
        log_error(f"新浪财经失败：{e}")
        time.sleep(0.5)
    
    # 3. 尝试第三数据源（腾讯财经）
    try:
        data = qq_http_get(code)
        if validate_data(data):
            return data, '腾讯财经'
    except Exception as e:
        log_error(f"腾讯财经失败：{e}")
    
    # 4. 所有数据源都失败 - 必须报告失败，禁止编造
    raise DataFetchError("所有数据源获取失败，无法提供数据")
```

## 📝 报告规范（必须遵守）

### 必须包含的信息

```markdown
## 数据来源
- **数据源**: [东方财富/新浪财经/腾讯财经]
- **获取时间**: YYYY-MM-DD HH:MM:SS
- **数据状态**: ✅ 实时 / ⚠️ 延迟 / ❌ 失败

## 数据质量说明
- 如使用备用数据源，必须说明原因
- 如有数据缺失，必须明确标注"待更新"
- 如有数据异常，必须提示用户
```

### 示例

```markdown
✅ 正确：
> 数据来源：新浪财经 HTTP API（东方财富限流，切换备用接口）

> 数据状态：✅ 实时

❌ 错误（违规）：
> 数据来源：东方财富  ← 实际是编造的
```

## ⚠️ 违规处理

### 违规行为

| 违规行为 | 严重程度 | 处理措施 |
|---------|---------|---------|
| **编造数据** | 🔴 严重 | 立即停止 + 报告用户 |
| **使用过期数据** | 🟠 中等 | 重新获取 + 警告 |
| **未标注数据源** | 🟡 轻微 | 补充标注 |
| **数据偏差>1%** | 🟠 中等 | 重新获取 + 说明 |

### 处理流程

```
发现违规
   ↓
立即停止当前操作
   ↓
报告用户并说明原因
   ↓
等待用户指示
   ↓
重新获取正确数据
```
