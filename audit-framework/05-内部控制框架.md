## 5.2 控制环境

### 核心要素
| 要素 | 内容 | 审计关注点 |
|------|------|------------|
| 诚信与道德价值观 | 行为准则、道德规范 | 是否有书面规范、是否执行 |
| 董事会独立性 | 独立董事比例、审计委员会 | 是否有效监督 |
| 组织结构 | 部门设置、汇报关系 | 职责是否清晰 |
| 权责分配 | 授权体系、岗位说明 | 是否匹配、有无越权 |
| 人力资源政策 | 招聘、培训、考核、晋升 | 是否公平、有效 |

### 控制环境评价清单
- [ ] 管理层是否强调诚信和道德行为
- [ ] 是否有书面的行为准则
- [ ] 董事会是否独立于管理层
- [ ] 审计委员会是否有效运作
- [ ] 组织结构是否清晰
- [ ] 权责分配是否合理
- [ ] 人力资源政策是否完善
- [ ] 员工是否理解并遵守规范

## 5.3 风险评估（内控视角）

### 内控相关风险
| 风险类型 | 示例 | 对应控制 |
|----------|------|----------|
| 外部风险 | 法规变化、市场波动 | 合规监控、应急预案 |
| 内部风险 | 人员流失、系统故障 | 继任计划、备份机制 |
| 流程风险 | 审批漏洞、职责冲突 | 职责分离、多级审批 |

### 舞弊风险因素
```
                    舞弊三角
                       │
         ┌─────────────┼─────────────┐
         │             │             │
         ▼             ▼             ▼
      压力/动机      机会          自我合理化
         │             │             │
    财务压力      控制薄弱       道德观念淡薄
    业绩压力      监督缺失       "大家都这样"
    个人问题      信息不对称     "公司欠我的"
```

### 舞弊风险审计程序
1. 访谈了解异常情况
2. 分析异常交易模式
3. 检查审批权限使用
4. 核实供应商/客户真实性
5. 盘点关键资产
6. 函证重要账户

## 5.4 控制活动

### 控制类型分类
```
控制活动
    │
    ├── 按功能分
    │   ├── 预防性控制（事前）
    │   │   └── 授权审批、职责分离、系统权限
    │   ├── 检查性控制（事中/事后）
    │   │   └── 复核、对账、盘点、分析
    │   └── 纠正性控制
    │       └── 差异调整、错误更正
    │
    ├── 按手段分
    │   ├── 人工控制
    │   ├── 自动化控制
    │   └── 混合控制
    │
    └── 按层级分
        ├── 公司层面控制
        ├── 流程层面控制
        └── 交易层面控制
```

### 关键控制活动

#### 1. 职责分离 (SoD)
**不相容职务**：
| 职务 A | 职务 B | 风险 |
|--------|--------|------|
| 授权 | 执行 | 越权操作 |
| 执行 | 记录 | 隐瞒错误 |
| 记录 | 保管 | 资产挪用 |
| 采购 | 验收 | 虚假采购 |
| 销售 | 收款 | 截留货款 |
| 出纳 | 会计 | 资金舞弊 |

**SoD 矩阵示例**：
```
         │ 申请 │ 审批 │ 执行 │ 验收 │ 付款 │
─────────┼──────┼──────┼──────┼──────┼──────│
 张三    │  ✓   │      │      │  ✓   │      │
 李四    │      │  ✓   │      │      │      │
 王五    │      │      │  ✓   │      │  ✓   │
```

#### 2. 授权审批
**授权类型**：
| 类型 | 描述 | 示例 |
|------|------|------|
| 一般授权 | 常规业务、标准权限 | 日常采购审批 |
| 特别授权 | 特殊业务、临时权限 | 重大投资审批 |
| 书面授权 | 正式文件明确 | 授权书、制度 |
| 口头授权 | 临时、紧急情况 | 需事后补书面 |

**审批权限表模板**：
| 业务类型 | 金额范围 | 审批人 | 备注 |
|----------|----------|--------|------|
| 采购 | <1 万 | 部门经理 | |
| 采购 | 1-10 万 | 分管副总 | |
| 采购 | 10-50 万 | 总经理 | |
| 采购 | >50 万 | 董事会 | |

#### 3. 实物控制
**控制措施**：

**应用控制**：

**评价指标**：

**异常调查**：

## 5.5 信息与沟通

### 信息系统
**关键要素**：
- [ ] 财务系统
- [ ] 业务系统
- [ ] 数据分析工具
- [ ] 报告系统

### 沟通机制
| 沟通类型 | 方向 | 方式 | 频率 |
|----------|------|------|------|
| 上行沟通 | 员工→管理层 | 报告、建议 | 定期/即时 |
| 下行沟通 | 管理层→员工 | 通知、培训 | 定期 |
| 平行沟通 | 部门间 | 会议、协作 | 按需 |
| 外部沟通 | 与外部 | 函件、会议 | 按需 |

### 沟通有效性评价
- [ ] 信息是否及时传递
- [ ] 信息是否准确完整
- [ ] 接收方是否理解
- [ ] 是否有反馈机制
- [ ] 是否有保密措施

## 5.6 监督活动

### 监督类型
```
监督活动
    │
    ├── 持续监督
    │   ├── 管理层日常监督
    │   ├── 系统自动监控
    │   └── 关键指标预警
    │
    └── 独立评价
        ├── 内部审计
        ├── 专项检查
        └── 外部审计
```

### 内控缺陷认定

#### 缺陷等级
| 等级 | 财务影响 | 定性标准 | 整改要求 |
|------|----------|----------|----------|
| 重大缺陷 | >利润 5% | 舞弊、财报重述 | 立即 |
| 重要缺陷 | 利润 3-5% | 关键控制失效 | 30 天 |
| 一般缺陷 | <利润 3% | 一般控制问题 | 60 天 |

#### 缺陷认定流程
```
发现缺陷 → 初步评估 → 详细分析 → 等级认定 → 报告 → 整改
   │          │          │          │        │      │
   ▼          ▼          ▼          ▼        ▼      ▼
 审计      影响       原因       集体      管理层   跟踪
 测试      判断       分析       讨论      审批    验证
```

### 内控评价报告
**报告内容**：
1. 评价范围和依据
2. 评价方法和程序
3. 缺陷认定标准
4. 发现的缺陷及整改
5. 内控有效性结论

## 5.7 控制测试方法

### 测试程序
| 程序 | 目的 | 适用场景 |
|------|------|----------|
| 询问 | 了解控制 | 所有控制 |
| 观察 | 验证实物控制 | 门禁、盘点 |
| 检查 | 验证文档控制 | 审批、记录 |
| 重新执行 | 验证控制有效性 | 关键控制 |

### 样本量参考
| 控制频率 | 年度发生次数 | 最小样本量 |
|----------|--------------|------------|
| 每日多次 | >250 | 25-60 |
| 每日一次 | 250 | 25-40 |
| 每周一次 | 52 | 5-15 |
| 每月一次 | 12 | 2-5 |
| 每季一次 | 4 | 1-2 |
| 每年一次 | 1 | 1 |

### 测试结果评价
| 偏差数 | 结论 | 后续行动 |
|--------|------|----------|
| 0 | 控制有效 | 可依赖 |
| 1 | 可能有效 | 扩大测试 |
| ≥2 | 控制无效 | 不可依赖、增加实质性测试 |
