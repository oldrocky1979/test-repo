# 第三章：审计流程（全周期闭环）

## 3.1 审计流程总览

```
┌─────────────────────────────────────────────────────────────────────────┐
│                         审计全流程图                                     │
│                                                                         │
│   年度规划    项目准备    现场执行    报告出具    整改跟踪    归档关闭    │
│      │           │           │           │           │           │      │
│      ▼           ▼           ▼           ▼           ▼           ▼      │
│  ┌───────┐  ┌───────┐  ┌───────┐  ┌───────┐  ┌───────┐  ┌───────┐     │
│  │风险   │  │立项   │  │测试   │  │起草   │  │整改   │  │资料   │     │
│  │评估   │→ │通知   │→ │取证   │→ │复核   │→ │计划   │→ │归档   │     │
│  │计划   │  │准备   │  │分析   │  │定稿   │  │验证   │  │总结   │     │
│  └───────┘  └───────┘  └───────┘  └───────┘  └───────┘  └───────┘     │

│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 3.2 阶段一：年度审计计划

### 输入
- 组织战略目标
- 上年度审计结果
- 风险评估结果
- 管理层要求
- 监管要求

### 流程步骤

#### Step 1: 风险评估（每年 Q4）
```
1. 识别风险领域
   ├─ 财务风险
   ├─ 运营风险
   ├─ 合规风险
   ├─ 战略风险
   └─ IT 风险

2. 评估风险等级
   │ 可能性 × 影响程度 = 风险值
   │
   ├─ 高 (红色)   → 必须审计
   ├─ 中 (黄色)   → 选择性审计
   └─ 低 (绿色)   → 暂不审计

3. 确定审计优先级
   └─ 按风险值排序
```

#### Step 2: 资源规划
| 资源类型 | 规划内容 | 考虑因素 |
|----------|----------|----------|
| 人力资源 | 人员数量、技能组合 | 项目需求、休假、培训 |
| 时间资源 | 审计人天 | 项目复杂度、地域分布 |
| 财务资源 | 审计预算 | 差旅、外包、工具 |
| 技术资源 | 审计软件、数据分析工具 | 系统许可、培训 |

#### Step 3: 计划编制
**年度审计计划书内容**：
1. 上年度审计工作总结
2. 本年度风险评估结果
3. 拟审计项目和优先级
4. 资源需求和预算
5. 关键时间节点

#### Step 4: 审批流程
```
首席审计官起草 → 审计委员会审议 → 董事会批准 → 正式下达
     │               │              │            │
   12 月初         12 月中旬        12 月底       1 月初
```

### 输出
- ✅ 《年度审计计划书》
- ✅ 《年度风险评估报告》
- ✅ 《审计资源预算表》

## 3.3 阶段二：项目准备

### 流程图
```
立项决策 → 组建团队 → 初步调查 → 编制方案 → 发送通知 → 进点准备
   │          │          │          │          │          │
   ▼          ▼          ▼          ▼          ▼          ▼
 审批      分工      了解      详细      提前      确认
          定责      业务      计划      3 日      就绪
```

### 关键活动

#### 1. 立项审批
**立项申请书内容**：
- 审计背景和目的
- 审计范围和重点
- 预计时间和资源
- 项目组成员

#### 2. 组建审计组
| 角色 | 职责 | 资质要求 |
|------|------|----------|
| 项目负责人 | 整体负责、报告审核 | 5 年 + 经验、CIA/CPA |
| 主审 | 现场管理、底稿复核 | 3 年 + 经验 |
| 审计员 | 执行测试、编制底稿 | 1 年 + 经验 |
| 专家支持 | 专业领域指导 | 相关专业资质 |

#### 3. 初步调查
**调查内容**：
- [ ] 被审计单位基本情况
- [ ] 组织架构和人员
- [ ] 主要业务流程
- [ ] 关键信息系统
- [ ] 以往审计发现
- [ ] 相关制度文件

**调查方法**：
- 资料收集（制度、报表、报告）
- 访谈（管理层、关键岗位）
- 数据分析（趋势、异常）
- 现场走访

#### 4. 编制审计方案
**审计方案内容**：
```markdown
1. 审计目标
2. 审计范围（时间范围、业务范围、组织范围）
3. 审计重点和风险领域
4. 审计程序和方法
5. 人员分工和时间安排
6. 需要被审计单位配合事项
```

#### 5. 发送审计通知书
**时间要求**：提前 3 个工作日送达

**通知书内容**：
- 审计依据
- 审计目的和范围
- 审计时间
- 审计组成员
- 需要准备的资料清单
- 联系人信息

### 输出
- ✅ 《审计立项申请书》
- ✅ 《审计方案》
- ✅ 《审计通知书》
- ✅ 《资料需求清单》

## 3.4 阶段三：现场执行

### 标准作业流程

```
┌────────────────────────────────────────────────────────────┐
│                    现场执行流程                             │
├────────────────────────────────────────────────────────────┤
│                                                            │
│  进点会议 → 控制测试 → 实质性测试 → 分析复核 → 底稿编制    │
│     │         │           │           │          │         │
│     ▼         ▼           ▼           ▼          ▼         │
│  明确      评价        获取        发现       记录         │
│  目的      有效性      证据        异常       轨迹         │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

### 1. 进点会议
**参会人员**：
- 审计组全体成员
- 被审计单位负责人
- 相关部门负责人
- 配合联络人

**会议议程**：
1. 介绍审计组成员
2. 说明审计目的和范围
3. 介绍审计方法和时间安排
4. 明确配合要求
5. 答疑交流

### 2. 控制测试
**测试目标**：评价内部控制设计和运行的有效性

**测试方法**：
| 方法 | 适用场景 | 样本量参考 |
|------|----------|------------|
| 询问 | 了解控制 | 关键人员 |
| 观察 | 实物控制、职责分离 | 现场观察 |
| 检查 | 文档控制、审批流程 | 25-60 个 |
| 重新执行 | 关键控制、自动化控制 | 10-25 个 |

**评价标准**：
- **设计有效**：控制能够防止/发现错报
- **运行有效**：控制一贯执行

### 3. 实质性测试
**测试类型**：
```
实质性测试
├── 细节测试
│   ├── 检查（凭证、合同、单据）
│   ├── 函证（银行、往来款）
│   ├── 盘点（现金、存货、固定资产）
│   └── 重新计算
│
└── 实质性分析程序
    ├── 趋势分析
    ├── 比率分析
    ├── 合理性测试
    └── 异常波动调查
```

**样本量确定**：
| 风险等级 | 覆盖率 | 样本量参考 |
|----------|--------|------------|
| 高 | 80-100% | 全查或大样本 |
| 中 | 50-80% | 中等样本 |
| 低 | 20-50% | 小样本 |

### 4. 审计取证
**证据要求**：充分、相关、可靠

**证据类型**：
| 类型 | 可靠性 | 示例 |
|------|--------|------|
| 外部证据 | 高 | 银行函证、供应商确认 |
| 内部证据（已验证） | 中 | 经审批的单据 |
| 内部证据（未验证） | 低 | 自制表格 |
| 口头证据 | 低 | 访谈记录（需佐证） |

**取证方法**：
- 复印/扫描（标注来源、日期、取证人员）
- 截图（系统数据，含时间戳）
- 拍照（实物资产、现场情况）
- 录音/录像（经同意，重要访谈）

### 5. 工作底稿
**底稿内容**：
```markdown
1. 底稿索引号
2. 审计项目名称
3. 审计事项
4. 审计期间
5. 编制人、编制日期
6. 复核人、复核日期
7. 审计目标
8. 审计程序
9. 审计发现
10. 审计结论
11. 附件清单
```

**底稿要求**：
- [ ] 内容完整、清晰
- [ ] 结论明确
- [ ] 证据充分
- [ ] 可追溯
- [ ] 签字齐全

**三级复核制度**：
```
审计员自核 → 主审复核 → 项目负责人审核
    │           │            │
  100%        关键         重大事项
            事项 100%
```

### 输出
- ✅ 《进点会议纪要》
- ✅ 《审计工作底稿》
- ✅ 《审计发现汇总表》
- ✅ 《问题确认单》

## 3.5 阶段四：报告出具

### 报告流程
```
起草报告 → 内部复核 → 征求意见 → 修改完善 → 正式出具 → 报送分发
   │          │          │          │          │          │
   ▼          ▼          ▼          ▼          ▼          ▼
 项目负责人  主审      被审计    沟通      签发       归档
 撰写        审核      单位      确认      审批       分发
            质控      (10 日)              (CAE)
```

### 审计报告结构
```markdown
# 审计报告

## 一、审计概况
- 审计依据
- 审计目的和范围
- 审计时间和方法
- 被审计单位基本情况

## 二、审计评价
- 总体评价
- 主要成绩

## 三、审计发现
- 问题描述（5W1H）
- 风险/影响分析
- 原因分析
- 审计建议

## 四、整改要求
- 整改责任人
- 整改期限
- 整改报告要求

## 五、附件
- 问题清单
- 相关证据
```

### 问题分级标准
| 级别 | 定义 | 示例 | 整改期限 |
|------|------|------|----------|
| 重大 | 可能导致重大损失或违规 | 舞弊、重大内控缺陷 | 立即 |
| 重要 | 可能影响目标实现 | 关键控制缺失 | 30 天 |
| 一般 | 需要改进但不紧急 | 流程瑕疵、文档不全 | 60 天 |
| 建议 | 优化建议 | 效率提升机会 | 90 天 |

### 报告质量要求
- **客观性**：基于事实，不夸大不缩小
- **清晰性**：语言简洁，逻辑清晰
- **及时性**：现场结束后 10 个工作日内
- **建设性**：建议可操作、有价值

### 输出
- ✅ 《审计报告（征求意见稿）》
- ✅ 《征求意见反馈表》
- ✅ 《审计报告（正式稿）》
- ✅ 《管理建议书》（如需要）

## 3.6 阶段五：整改跟踪

### 跟踪流程
```
┌─────────────────────────────────────────────────────────────┐
│                    整改跟踪闭环                              │
│                                                             │
│  问题移交 → 整改计划 → 过程监控 → 整改验证 → 关闭/升级      │
│     │          │          │          │          │          │
│     ▼          ▼          ▼          ▼          ▼          │
│   登记      制定      定期      现场      完成         上报  │
│   台账      方案      跟进      确认      关闭         督办  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1. 问题移交
**整改通知单内容**：
- 问题描述
- 风险等级
- 整改建议
- 整改责任人
- 整改期限
- 验收标准

### 2. 整改计划
**被审计单位提交**：
- 整改措施（具体行动）
- 责任人和配合人
- 时间节点（里程碑）
- 所需资源
- 预期效果

### 3. 过程监控
**监控频率**：
| 问题等级 | 监控频率 | 方式 |
|----------|----------|------|
| 重大 | 每周 | 现场 + 报告 |
| 重要 | 每两周 | 报告 + 沟通 |
| 一般 | 每月 | 报告 |

**监控内容**：
- 整改进度
- 遇到的困难
- 需要的支持
- 预计完成时间

### 4. 整改验证
**验证方法**：
- 文档审查（制度、流程、记录）
- 现场观察（控制执行）
- 人员访谈（了解变化）
- 数据测试（验证效果）
- 重新执行（测试控制）

**验证标准**：
| 状态 | 标准 |
|------|------|
| 已完成 | 措施落实、效果达成、证据充分 |
| 进行中 | 按计划推进、无重大偏差 |
| 延期 | 有合理原因、有新计划 |
| 未完成 | 无进展或进展缓慢 |

### 5. 关闭/升级
**关闭条件**：
- [ ] 整改措施已落实
- [ ] 整改效果已验证
- [ ] 相关证据已归档
- [ ] 审计组确认

**升级机制**：
```
首次延期 → 提醒责任人
二次延期 → 报告管理层
三次延期 → 报告审计委员会
拒不整改 → 问责处理
```

### 输出
- ✅ 《整改跟踪台账》
- ✅ 《整改进度报告》
- ✅ 《整改验证报告》
- ✅ 《问题关闭确认单》

## 3.7 阶段六：归档关闭

### 归档内容
| 类别 | 文件清单 |
|------|----------|
| 立项文件 | 立项申请、审计方案、通知书 |
| 过程文件 | 会议纪要、底稿、取证材料 |
| 报告文件 | 报告草稿、征求意见、正式报告 |
| 整改文件 | 整改计划、验证报告、关闭确认 |
| 其他文件 | 项目总结、经验教训 |

### 归档要求
- **完整性**：所有文件齐全
- **规范性**：格式统一、签字齐全
- **及时性**：项目结束后 1 个月内
- **保密性**：按密级管理

### 项目总结
**总结内容**：
1. 项目目标达成情况
2. 主要发现和成果
3. 经验和教训
4. 改进建议（对审计方法）
5. 后续关注事项

### 知识管理
- 典型案例入库
- 审计程序优化
- 检查清单更新
- 培训材料整理

### 输出
- ✅ 《审计档案》
- ✅ 《项目总结报告》
- ✅ 《经验教训记录》

## 3.8 流程关键控制点

| 阶段 | 控制点 | 控制措施 |
|------|--------|----------|
| 计划 | 风险评估 | 多维度评估、管理层确认 |
| 准备 | 独立性审查 | 利益冲突申报、回避 |
| 执行 | 质量控制 | 三级复核、底稿审核 |
| 报告 | 事实确认 | 问题确认单、签字 |
| 整改 | 跟踪验证 | 现场验证、证据收集 |
| 归档 | 完整性检查 | 归档清单、逐项核对 |
