# 第四章：风险评估方法

## 4.1 风险评估框架

### 总体框架
```
┌─────────────────────────────────────────────────────────────┐
│                    风险评估流程                              │
│                                                             │
│   目标设定 → 风险识别 → 风险分析 → 风险评价 → 风险应对      │
│      │          │          │          │          │          │
│      ▼          ▼          ▼          ▼          ▼          │
│   明确       找出       评估       排序       制定         │
│   方向       风险       影响       优先级     策略         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 4.2 目标设定

### 目标层级
```
组织战略目标
      │
      ▼
经营目标 ──→ 运营效率、效果
      │
      ▼
报告目标 ──→ 财务报告可靠性
      │
      ▼
合规目标 ──→ 法律法规遵循
```

### 目标设定原则（SMART）
- **S**pecific 具体
- **M**easurable 可衡量
- **A**chievable 可实现
- **R**elevant 相关
- **T**ime-bound 时限

        │                  │                  │

        │                  │                  │

        ├─ 竞争加剧        ├─ 系统故障        ├─ 财务报告

### 识别方法

#### 1. 头脑风暴法
**适用场景**：新业务、新领域、全面风险评估

**操作步骤**：
1. 准备阶段
   - 确定参会人员（跨部门、跨层级）
   - 准备背景资料
   - 设定讨论框架

2. 讨论阶段
   - 自由发言、不批评
   - 记录所有观点
   - 鼓励发散思维

3. 整理阶段
   - 归类整理
   - 去重合并
   - 形成风险清单

#### 2. 流程图法
**适用场景**：流程风险识别

**操作步骤**：
```
绘制流程图 → 识别关键节点 → 分析节点风险 → 记录风险点
     │            │              │            │
   业务        决策点          可能        风险描述
   流程        控制点          问题        及影响
```

#### 3. 检查清单法
**适用场景**：常规风险、合规风险

**示例清单**：
- [ ] 是否有未经授权的访问？
- [ ] 是否有职责分离冲突？
- [ ] 是否有超权限审批？
- [ ] 是否有异常交易？
- [ ] 是否有文档缺失？

#### 4. 访谈法
**适用场景**：深入了解特定风险

**访谈对象**：
| 层级 | 关注重点 |
|------|----------|
| 高管层 | 战略风险、重大决策 |
| 中层管理 | 流程风险、执行问题 |
| 基层员工 | 操作风险、日常问题 |

**访谈技巧**：
- 开放式问题为主
- 追问具体事例
- 注意非语言信息
- 记录关键原话

#### 5. 数据分析法
**适用场景**：财务风险、运营风险

**分析方法**：
- 趋势分析（同比、环比）
- 比率分析（财务比率、运营指标）
- 异常检测（超出阈值、偏离常态）
- 关联分析（数据间关系）

## 4.4 风险分析

### 可能性评估
| 等级 | 描述 | 定量参考 | 定性描述 |
|------|------|----------|----------|
| 5-极高 | 几乎确定发生 | >80% | 每年多次发生 |
| 4-高 | 很可能发生 | 60-80% | 每年发生 |
| 3-中 | 可能发生 | 40-60% | 每 2-3 年发生 |
| 2-低 | 不太可能发生 | 20-40% | 每 5 年发生 |
| 1-极低 | 极少发生 | <20% | 10 年以上 |

### 影响程度评估
| 等级 | 财务影响 | 运营影响 | 声誉影响 | 合规影响 |
|------|----------|----------|----------|----------|
| 5-极高 | >1000 万 | 业务中断>1 周 | 全国性负面 | 刑事责任 |
| 4-高 | 500-1000 万 | 业务中断 3-7 天 | 行业负面 | 重大处罚 |
| 3-中 | 100-500 万 | 业务中断 1-3 天 | 局部负面 | 一般处罚 |
| 2-低 | 50-100 万 | 业务中断<1 天 | 轻微影响 | 警告 |
| 1-极低 | <50 万 | 无影响 | 无影响 | 无影响 |

### 风险矩阵
```
                    影响程度
              1    2    3    4    5
            ┌────┬────┬────┬────┬────┐
          5 │ 中 │ 高 │ 高 │ 极高│极高│
            ├────┼────┼────┼────┼────┤
          4 │ 低 │ 中 │ 高 │ 高 │ 极高│
    可      ├────┼────┼────┼────┼────┤
    能      3 │ 低 │ 低 │ 中 │ 高 │ 高 │
    性      ├────┼────┼────┼────┼────┤
          2 │ 低 │ 低 │ 低 │ 中 │ 高 │
            ├────┼────┼────┼────┼────┤
          1 │ 低 │ 低 │ 低 │ 低 │ 中 │
            └────┴────┴────┴────┴────┘
```

### 风险值计算
```
风险值 = 可能性 × 影响程度

风险等级划分：
- 极高风险：20-25 分 → 立即处理
- 高风险：15-19 分 → 优先处理
- 中风险：8-14 分 → 计划处理
- 低风险：1-7 分 → 接受/监控
```

## 4.5 风险评价

### 评价维度
```
                    风险评价
                       │
         ┌─────────────┼─────────────┐
         │             │             │
     固有风险      控制有效性      剩余风险
         │             │             │
    未考虑控制    控制测试        考虑控制后
    前的风险      结果            的风险
```

### 固有风险评估
评估不考虑任何控制措施时的风险水平

**考虑因素**：
- 业务复杂性
- 交易金额大小
- 人员素质
- 外部环境
- 历史问题

### 控制有效性评估
| 等级 | 描述 | 测试结论 |
|------|------|----------|
| 有效 | 控制设计合理且一贯执行 | 可依赖 |
| 部分有效 | 控制设计合理但执行不稳定 | 有限依赖 |
| 无效 | 控制缺失或严重缺陷 | 不可依赖 |

### 剩余风险评估
```
剩余风险 = 固有风险 × (1 - 控制有效性)

示例：
固有风险：高 (4 分)
控制有效性：70%
剩余风险 = 4 × (1 - 0.7) = 1.2 分 → 低风险
```

## 4.6 风险应对策略

### 四种策略
```
┌────────────────────────────────────────────────────────────┐
│                    风险应对策略                             │
├─────────────┬─────────────┬─────────────┬─────────────────┤
│   风险规避   │   风险降低   │   风险分担   │   风险承受     │
├─────────────┼─────────────┼─────────────┼─────────────────┤
│ 停止高风险  │ 加强控制    │ 购买保险    │ 接受风险        │
│ 业务活动    │ 改进流程    │ 外包        │ 成本>收益       │
│ 退出市场    │ 技术升级    │ 合作分担    │ 在风险偏好内    │
└─────────────┴─────────────┴─────────────┴─────────────────┘
```

### 策略选择矩阵
| 风险等级 | 推荐策略 | 示例 |
|----------|----------|------|
| 极高 | 规避/降低 | 停止违规业务、立即整改 |
| 高 | 降低/分担 | 加强控制、购买保险 |
| 中 | 降低/承受 | 改进流程、监控 |
| 低 | 承受 | 接受、定期复核 |

### 风险登记册
完整的风险登记册应包含：
1. 风险基本信息（编号、名称、类别）
2. 风险描述（原因、事件、后果）
3. 风险评估结果（可能性、影响、等级）
4. 风险责任人
5. 应对措施和计划
6. 监控指标和频率
7. 状态更新记录

### 关键风险指标 (KRI)
| 风险类型 | KRI 示例 | 阈值 | 预警级别 |
|----------|----------|------|----------|
| 资金风险 | 现金周转天数 | >60 天 | 黄色 |
| 信用风险 | 应收账款逾期率 | >10% | 红色 |
| 运营风险 | 订单交付及时率 | <95% | 黄色 |
| 合规风险 | 监管处罚次数 | >0 | 红色 |
| IT 风险 | 系统可用率 | <99.5% | 红色 |

## 4.8 风险评估频率

| 评估类型 | 频率 | 触发条件 | 参与人员 |
|----------|------|----------|----------|
| 全面评估 | 年度 | 固定 | 全员参与 |
| 专项评估 | 按需 | 新业务、重大变化 | 相关人员 |
| 持续监控 | 月度/季度 | 固定 | 风险责任人 |
| 临时评估 | 即时 | 突发事件 | 应急小组 |
