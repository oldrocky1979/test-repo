# A 股数据获取规则

## 📋 总则

**生效日期**: 2026-03-13  
**适用范围**: 所有 A 股数据获取场景  
**执行要求**: 必须严格执行，不得违反

## 🚫 核心原则（红线）

### 第一条：禁止编造数据 ⚠️
- ❌ **绝对禁止**猜测、估算、编造任何股票数据
- ❌ **绝对禁止**使用缓存数据冒充实时数据
- ❌ **绝对禁止**在数据缺失时自行填充
- ✅ **必须**标注数据来源和获取时间
- ✅ **必须**如实报告数据获取失败

### 第二条：数据源优先级 📊

```
1️⃣ 东方财富 (AkShare) - 主数据源
   ↓ (失败/限流时切换)
2️⃣ 新浪财经 HTTP API - 备用数据源
   ↓ (失败/限流时切换)
3️⃣ 腾讯财经 HTTP API - 第三数据源
```

### 第三条：数据验证 ✅
- ✅ 关键数据必须交叉验证（如：今开、最高、最低、最新价）
- ✅ 发现数据异常立即报告用户
- ✅ 数据偏差超过 1% 必须重新获取

## 🔧 数据获取流程

### 标准流程

```python
def get_stock_data(code):
    """
    获取股票数据 - 严格遵守数据获取规则
    """
    
    # 1. 尝试主数据源（东方财富）
    try:
        data = akshare_get(code)
        if validate_data(data):  # 数据验证
            return data, '东方财富'
    except Exception as e:
        log_error(f"东方财富失败：{e}")
        time.sleep(0.5)  # 等待后切换
    
    # 2. 尝试备用数据源（新浪财经）
    try:
        data = sina_http_get(code, delay=0.3)
        if validate_data(data):
            return data, '新浪财经'
    except Exception as e:
        log_error(f"新浪财经失败：{e}")
        time.sleep(0.5)
    
    # 3. 尝试第三数据源（腾讯财经）
    try:
        data = qq_http_get(code)
        if validate_data(data):
            return data, '腾讯财经'
    except Exception as e:
        log_error(f"腾讯财经失败：{e}")
    
    # 4. 所有数据源都失败
    raise DataFetchError(f"所有数据源获取失败，无法编造数据")
```

### 数据验证规则

```python
def validate_data(data):
    """验证数据有效性"""
    
    # 必填字段检查
    required_fields = ['open', 'high', 'low', 'price', 'volume']
    for field in required_fields:
        if field not in data or data[field] is None:
            return False
    
    # 合理性检查
    if not (data['low'] <= data['price'] <= data['high']):
        return False
    
    if data['open'] < data['low'] or data['open'] > data['high']:
        return False
    
    # 涨跌停检查（A 股±10%）
    if data['change_pct'] > 11 or data['change_pct'] < -11:
        return False
    
    return True
```

## ⚠️ 违规处理

### 违规行为定义

| 违规行为 | 严重程度 | 示例 |
|---------|---------|------|
| **编造数据** | 🔴 严重 | 接口失败后自行填充数据 |
| **使用过期数据** | 🟠 中等 | 用昨天的数据冒充今天 |
| **未标注数据源** | 🟡 轻微 | 报告中不写数据来源 |
| **数据偏差>1%** | 🟠 中等 | 与实际情况偏差超过 1% |

### 处理措施

1. **首次违规** - 立即更正 + 书面警告
2. **二次违规** - 暂停功能 + 重新培训
3. **三次违规** - 永久禁用 + 报告用户
