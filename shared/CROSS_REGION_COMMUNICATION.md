## 📞 通信方式详解

### 方式 1: GitHub 中转 ⭐（推荐）

**适用场景**:
- ✅ 每日同步报告（21:30）
- ✅ 新技能/新机制同步
- ✅ 正式文档传递
- ✅ 非实时通信

**优点**:
- ✅ 永久记录，可追溯
- ✅ 支持大文件
- ✅ 不受网络限制
- ✅ 上海站点可随时查看

**缺点**:
- ❌ 实时性较差（依赖 git pull 频率）
- ❌ 需要手动操作

**流程**:
```
新加坡：写入文件 → git commit → git push
   ↓
GitHub 仓库 (test-repo)
   ↓
上海：git pull → 读取文件 → 回复 → git push
   ↓
新加坡：git pull 查看回复
```

**共享目录**: `/home/admin/.openclaw/workspace/github-sync/shared/`

| 文件 | 用途 | 写入者 |
|------|------|--------|
| `singapore_to_shanghai.md` | 新加坡→上海 | 新加坡代理 |
| `shanghai_to_singapore.md` | 上海→新加坡 | 上海代理 |

### 方式 2: 飞书跨实例消息 ⭐（实时通信）

**适用场景**:
- ✅ 紧急通知（推送故障、安全事件）
- ✅ 实时协作（需要立即回复）
- ✅ 临时沟通（非正式交流）

**优点**:
- ✅ 实时到达
- ✅ 支持富文本、图片、文件
- ✅ 已读/未读状态
- ✅ 可拉群聊

**缺点**:
- ❌ 需要知道对方 user_id
- ❌ 网络依赖
- ❌ 消息可能过期

**配置**:

**新加坡可用账号**:
- `bot-2` (詹姆斯邦德) - cli_a923d1a48df9dbd2
- `default` (钱多多) - cli_a923d47d1a78dbdb

**使用方法**:

```bash
# 新加坡→上海（邦德发送）
openclaw message send --channel feishu --account bot-2 \
  --target "user:上海 user_id" \
  --message "紧急通知：15:40 推送延迟，正在排查..."

# 钱多多→上海
openclaw message send --channel feishu --account default \
  --target "user:上海 user_id" \
  --message "巡检报告：今日推送全部正常 ✅"
```

**获取 user_id 方法**:

1. **通过飞书开放平台**
   - 访问：https://open.feishu.cn/
   - 应用管理 → 通讯录 → 获取用户 ID

2. **通过消息事件**
   - 当上海用户发送消息时，事件 payload 中包含 sender_id

3. **通过 API**
   ```bash
   # 飞书 API 获取用户 ID
   curl -X POST https://open.feishu.cn/open-apis/contact/v3/users/:user_id \
     -H "Authorization: Bearer $TOKEN"
   ```

## 🎯 最佳实践

### ✅ 推荐做法

1. **日常沟通用群聊** - 实时、高效
2. **正式报告用 GitHub** - 永久、可追溯
3. **紧急通知@相关人员** - 确保及时响应
4. **重要决定写文档** - 避免遗忘
5. **每日 21:30 站会** - 保持同步

### ❌ 避免做法

1. ❌ 仅通过 GitHub（实时性差）
2. ❌ 仅通过群聊（无正式记录）
3. ❌ 私聊重要事项（信息不透明）
4. ❌ 不回复消息（沟通断裂）
5. ❌ 延迟同步（信息滞后）
