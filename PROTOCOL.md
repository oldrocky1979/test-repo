**版本：** 1.0  
**创建时间：** 2026-03-05  
**适用范围：** 主代理 + 所有子代理

## 核心回复规则

### 标准工作流程

```
用户发消息 
  ↓
【1】立即秒回 (1-3 秒内)
  ↓
【2】理解任务 + 评估用时
  ↓
【3】分配子代理执行 (如需要)
  ↓
【4】执行完成 → 汇报结果
```

### 详细规范

#### 1️⃣ 秒回阶段
**目标：** 让用户知道消息已收到，代理在线

**回复示例：**
- "收到，我看看"
- "好的，马上处理"
- "明白，稍等"
- "收到，评估一下"

**要求：**
- 必须在 1-3 秒内回复
- 简短、自然、有人情味
- 不要长篇大论

---

#### 2️⃣ 评估阶段
**目标：** 理解任务，评估复杂度，决定执行策略

**检查清单：**
- [ ] 任务类型是什么？（查询/执行/创建/分析）
- [ ] 需要多少步骤？
- [ ] 是否需要子代理？
- [ ] 预计用时多久？
- [ ] 有什么风险或依赖？

**回复示例：**
- "这个需要检查 3 个地方，大约 2 分钟"
- "需要调用 API，预计 30 秒"
- "这个任务可以立即完成"

---

#### 3️⃣ 分配阶段
**目标：** 选择最合适的执行方式

**决策树：**
```
简单任务 → 主代理直接执行
复杂任务 → spawn 子代理
需要代码 → spawn ACP 子代理
需要等待 → 后台执行 + 轮询
```

**子代理类型：**
- `runtime: "subagent"` - 通用子代理
- `runtime: "acp"` - 代码/开发任务

---

#### 4️⃣ 汇报阶段
**目标：** 任务完成后给用户清晰的结果

**汇报结构：**
```
✅ 任务完成 / ❌ 遇到问题

**结果：**
[具体结果/输出]

**详情：**
- 关键点 1
- 关键点 2

**下一步：**
[建议的后续操作，如有]
```

## 记忆系统规则

### 文件结构
```
workspace/
├── PROTOCOL.md          # 本文件（核心规则）
├── SESSION-STATE.md     # 热内存（活跃上下文）
├── MEMORY.md            # 长期记忆（ curated）
└── memory/
    └── YYYY-MM-DD.md    # 每日日志
```

### 记录规则

**必须记录的：**
- ✅ 用户明确说"记住这个"
- ✅ 重要决策/偏好
- ✅ 项目关键信息
- ✅ 教训/错误（避免重复）

**可选记录的：**
- 日常对话流水
- 临时上下文

## 子代理配置模板

### 通用子代理
```javascript
sessions_spawn({
  task: "具体任务描述",
  runtime: "subagent",
  mode: "run",  // 或 "session"
  label: "任务标签"
})
```

### ACP 代码代理
```javascript
sessions_spawn({
  task: "代码任务描述",
  runtime: "acp",
  agentId: "codex",  // 或其他 ACP agent
  mode: "session",
  thread: true
})
```

## 紧急情况处理

### 代理无响应
1. 检查 `openclaw gateway status`
2. 检查子代理列表 `subagents action=list`
3. 重启网关 `openclaw gateway restart`

### 记忆丢失
1. 从 git 恢复：`git checkout .`
2. 从云端下载备份
3. 检查 `memory/` 目录
