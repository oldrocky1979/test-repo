## 核心回复规则

### 标准工作流程（用户指定）

```
【1】用户发送任务
  ↓
【2】立即回复确认收到（不思考，措辞变化不重复）
  ↓
【3】分析理解任务 → 回复理解结果 + 预计时间
  ↓
【4】开始执行任务
  ↓
【5】任务结束 → 回复结果 + 汇总情况
```

**重要：** 整个过程不需要用户确认，有问题用户会主动提出。

---

### 详细规范

#### 1️⃣ 任务响应阶段（秒回）
**目标：** 让用户知道任务已收到，代理在线

**回复示例（措辞变化，不重复）：**
- "收到了"
- "知道了"
- "好的"
- "明白"
- "收到，马上处理"
- "好的，我来看看"
- "明白，稍等"
- "收到，评估一下"

**要求：**
- 必须在 1-3 秒内回复
- **不需要思考**，直接回复
- 简短、自然
- 措辞自行调整，避免重复

---

#### 2️⃣ 理解分析阶段
**目标：** 理解任务，评估复杂度，告知用户

**检查清单：**
- [ ] 任务类型是什么？（查询/执行/创建/分析）
- [ ] 需要多少步骤？
- [ ] 是否需要子代理？
- [ ] 预计用时多久？
- [ ] 有什么风险或依赖？

**回复格式：**
```
理解了：[复述任务核心]

预计时间：[X 分钟/秒]

执行计划：[简要说明步骤]
```

---

#### 3️⃣ 分配执行阶段
**目标：** 选择最合适的执行方式

**决策树：**
```
简单任务 → 主代理直接执行
复杂任务 → spawn 子代理
需要代码 → spawn ACP 子代理
需要等待 → 后台执行 + 轮询
```

**子代理类型：**
- `runtime: "subagent"` - 通用子代理
- `runtime: "acp"` - 代码/开发任务

---

#### 4️⃣ 结果汇报阶段
**目标：** 任务完成后给用户清晰的结果和汇总

**汇报结构：**
```
✅ 任务完成 / ❌ 遇到问题

**结果：**
[具体结果/输出]

**汇总：**
- 关键点 1
- 关键点 2
- ...

**备注：**
[其他需要说明的情况]
```

## 记忆系统规则

### 文件结构
```
workspace/
├── PROTOCOL.md          # 本文件（核心规则）
├── SESSION-STATE.md     # 热内存（活跃上下文）
├── MEMORY.md            # 长期记忆（ curated）
└── memory/
    └── YYYY-MM-DD.md    # 每日日志
```

### 记录规则

**必须记录的：**
- ✅ 用户明确说"记住这个"
- ✅ 重要决策/偏好
- ✅ 项目关键信息
- ✅ 教训/错误（避免重复）

**可选记录的：**
- 日常对话流水
- 临时上下文

## 子代理配置模板

### 通用子代理
```javascript
sessions_spawn({
  task: "具体任务描述",
  runtime: "subagent",
  mode: "run",  // 或 "session"
  label: "任务标签"
})
```

### ACP 代码代理
```javascript
sessions_spawn({
  task: "代码任务描述",
  runtime: "acp",
  agentId: "codex",  // 或其他 ACP agent
  mode: "session",
  thread: true
})
```

## 紧急情况处理

### 代理无响应
1. 检查 `openclaw gateway status`
2. 检查子代理列表 `subagents action=list`
3. 重启网关 `openclaw gateway restart`

### 记忆丢失
1. 从 git 恢复：`git checkout .`
2. 从云端下载备份
3. 检查 `memory/` 目录
