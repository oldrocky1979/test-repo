# Skills 发现与评估流程 v2.0

**创建日期**: 2026-03-15  
**核心原则**: **先评估，后安装！绝不直接安装！**

---

## 🚫 红线规则

### ❌ 绝对禁止
- ❌ **找到 skill 后直接安装**
- ❌ **未经评估就执行**
- ❌ **忽略安全警告**
- ❌ **运行未审查的代码**
- ❌ **跳过安全扫描**

### ✅ 必须遵守
- ✅ **所有 skill 必须先评估**
- ✅ **评估报告必须存档**
- ✅ **高风险 skill 必须人工审核**
- ✅ **安装前必须用户确认**
- ✅ **安装前必须安全扫描** ⭐ 新增

## 🔍 发现渠道详解

### 渠道 1: Twitter/X（前沿动态）⭐

**为什么重要**：
- 开发者第一时间发布新项目
- 可以看到开发者的技术背景
- 了解社区反馈和讨论

**搜索策略**：
```
# 关键词搜索
"openclaw skill" filter:links
"clawdbot extension" filter:links
"ai agent tool" github filter:links

# 关注关键人物
- @openclaw_dev
- @clawdbot
- AI/自动化领域开发者

# 时间范围
- 最近 7 天（最新）
- 最近 30 天（热门）
```

**操作流程**：
1. 搜索 Twitter 找到相关发推
2. 查看推主背景（是否可信开发者）
3. 点击 GitHub 链接到项目页面
4. 阅读 README 和文档
5. 进入评估流程

---

### 渠道 2: GitHub 全网搜索（主要来源）⭐⭐⭐

**搜索词**：
```bash
# 核心搜索
openclaw skill agent

# 按热度排序

# 按时间排序（发现最新）
```

**检查项目**：

- ⚠️ Issues 数量和质量
- 📅 最后更新时间（<3 个月）
- 👤 开发者其他项目
- 📄 README 完整性

## 📊 评估标准详解

### ⭐ 实用性评估（40%）

| 得分 | 标准 |
|------|------|
| 5.0 | 解决高频痛点，不可替代，使用场景广泛 |
| 4.0 | 有明确用途，场景较广，有一定价值 |
| 3.0 | 特定场景有用，但可替代 |
| 2.0 | 用途有限，价值不高 |
| 1.0 | 鸡肋功能，无实际价值 |

**评估问题**：
- 这个 skill 解决什么问题？
- 问题是否高频/痛点？
- 是否有更好的替代方案？
- 使用场景是否广泛？

---

### 🔒 安全性评估（40%）⭐⭐⭐

**检查清单**：

#### 代码审查
- [ ] 是否有 `rm -rf` 等危险命令？
- [ ] 是否有 `curl | bash` 等管道执行？
- [ ] 是否有未解释的 `exec` 调用？
- [ ] 是否有文件读写操作？权限范围？
- [ ] 是否有网络请求？发送到哪里？
- [ ] 是否读取环境变量/配置文件？
- [ ] 是否有硬编码的 API key/密码？

#### 权限审查
- [ ] 需要哪些 OpenClaw 工具权限？
- [ ] 权限是否超出功能需求？
- [ ] 是否有 `elevated` 提权操作？
- [ ] 是否访问敏感数据（消息、文件）？

#### 依赖审查
- [ ] 依赖是否过多？
- [ ] 依赖是否可信？
- [ ] 是否有已知安全漏洞？

**安全等级**：
- ✅ **已验证** - 代码清晰，无危险操作，权限合理
- ⚠️ **需审查** - 有敏感操作但有注释说明，需人工确认
- ❌ **高风险** - 有未解释的危险操作，不建议安装
